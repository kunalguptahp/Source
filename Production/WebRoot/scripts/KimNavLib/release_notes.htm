1. TD style declaration in header.css should be moved into the header_body.css (this should resolve the issues with the SP banner top row)

ColorModuleHeaderArrow seems to be obsolete. Presently ColorModuleHeader also seems to be unused by the only ipg module, which refers to it.


3. NavModuleTitle and NavModuleSubCategory styles are in homepages-v5.css but should be in · Header_hpr.css

NavModuleTitle appears in as many as 12 css files in the library. Most Jsp pages seem to include the homepages-v5.css but the actual class gets called from banner_color_hpr.css. A complete overhaul of the CSS folder is required to resolve this anomaly.

4. All the table styles in homepages-v5.css should be in the header_body.css

Only generic tags should be present in the header_body.css, homepages-v5.css file's specific classes cannot be arbitrarily moved into header_body.css

New Functions

This section lists all of the functions that are new in Version 5.0 of header.js.

 

Site Search

setSiteSearch( sSID, Optional sLabelTitle )

           

Parameters:

            sSID

This value is your SiteSearch ID, which is created by the SiteSearch tool.  Entering an incorrect value can cause erroneous results or failure on the Search Results Page

 

                        sLabelTitle

Optional, default value is ‘This Site’.  Enter a title you want the customer base to see 

 

            Comments:

You should only turn this on if you have registered with the Search team for SiteSearch functionality.  This will place 2 radio buttons in the banner with the default set to search your site.

 

            Example:

Add ability for users to search your site.  Label the radio button with “My Sample Site”.  The site is registered with ID=000015.

 

            Implementation:

                                <script language="javascript" src="header.js"></script>

<script language="javascript">

setBanner("My Sample Site");

setSiteSearch("000015", "My Search");

                                                drawHeader();

                                </script>

 

                Risks:

                Entering a long title will cause the text to wrap and the banner to increase in size.

 

 

 

Display Login

displayLogin(Optional sURL)

 

Parameters:

                        sURL

Optional, default will take the user to the portal upon logging in. Enter a URL to a protected page on your site.        

           

Comments:

This function will display a Login button above the PeopleFinder text box.  The login button will send the user to the protected page.  Siteminder should protect this page to force the user to login in.  The login form allows them to login via SEA or Digital Badge. 

 

            Example:

Turn on the “Log On” button.  When pressed send the user to https://directory.hp.com/rfu/.

 

            Implementation:

 

                                <script language="javascript" src="header.js"></script>

<script language="javascript">

setBanner("My Sample Site");

                                                setSiteSearch("000015", "My Search");

                                                displayLogin(“https://directory.hp.com/rfu/”);

                                                drawHeader();

                                </script>

 

 

Use Horizontal Navigation

setHorizontalNav();

 

Comments: By default the 2nd level navigation is a vertical dropdown (classic mode).  By setting this function prior to the creation of the navigation elements, causes the 2nd level navigation to be a horizontal bar.

 

            Example:

 

                                Top_Nav.js

 

                                setHorizontalNav();

createMenu("menu1", "Menu 1", "<add absolute url here>","help");

                                               

                                createMenu("menu2", "Menu 2", "<add absolute url here>","help");

                                addMenuItem("menu2", "Sub-category 1", "http://us.hr.hp.com/benefits");

                                addMenuItem("menu2", "Sub-category 2", "http://us.hr.hp.com/benefits");

                                addMenuItem("menu2", "Sub-category 3", "http://us.hr.hp.com/benefits");

 

           

This causes ‘Sub-category 1’ and ‘Sub-category 2’ to be displayed horizontally on a line below the top level navigation

 

In the example above this will force all pages calling Top_Nav.js to have 2nd level horizontal navigation.  If you want to manually set some pages to use 2nd level horizontal and others to have vertical dropdown.  Then you need to call the function prior to including the Top_Nav.js.

 

e.g.

<script language="javascript" src="header.js"></script>

<script language="javascript">

                setHorizontalNav();

                                </script>

<script language="javascript" src="Top_Nav.js"></script>

 

 

Changing The Version Number

 

setVersionNumber( sVersionNumber )

 

Parameters:

                        sVersionNumber

Set the version number of your site.  

 

Comments:

The default version number displayed is the @hp portal version.

 

If you set the version number the text will change from ‘@hp Portal Version’ to ‘Site Version’

 

You must call this before calling the drawFooter() function.

 

            Example:

Have the footer display “Site Virsion 2.1” in the footer instead of the @hp footer version.

 

            Implementation:

 

                                <script language="javascript" src="header.js"></script>

<script language="javascript">

setBanner("My Sample Site");

                                                setSiteSearch("000015", "My Search");

                                                displayLogin(“https://directory.hp.com/rfu/”);

                                                drawHeader();

                                                setVersionNumber(“2.1”)

                                                drawFooter();

                                </script>
